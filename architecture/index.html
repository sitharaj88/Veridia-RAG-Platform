System Architecture & Flow Design

The RAG Intelligence platform is structured as a decoupled **Option A (Hybrid Full-Stack)** architecture. This separates high-speed user interface execution from intensive machine learning model execution and vector storage.

Low-Level Design (LLD) & Data Flows

Select a tab below to inspect how data passes step-by-step through each pipeline within the Python RAG core packages:

When documents are uploaded via POST /api/ingest, they progress through the structural loaders and contextual summary generator:

1
1. File Loadingsrc/ingestion/loader.py

Accepts PDFs, DOCX, MD, or developer code. Normalizes paragraph splits and returns raw text content.

2
2. Scanned OCR Fallbackrapidocr-onnxruntime

If extracted text length is 0, converts pages to images and extracts layout text with local ONNX models.

3
3. Document Context Injectorsrc/ingestion/chunker.py (Contextual)

Takes the first 4,000 characters of the loaded document and calls Ollama to build a 2-sentence overview summary.

4
4. Recursive Chunker Splitsrc/ingestion/chunker.py

Divides document text recursively based on character delimiters. Prepends the global summary to the start of each text block.

5
5. Embedding & Vector Indexingsrc/vectorstore/chroma_store.py

Embeds each contextual block with SentenceTransformers (384-dimensions) and persists them in ChromaDB collection collections.

Decoupling Benefit: By placing this entire multi-stage LLD core in the Python FastAPI layer, the Next.js frontend has a clean and simple interface, querying only the `/api/query/stream` endpoint for SSE streams.
