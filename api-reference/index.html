API Reference

REST API Specifications

The Aetheris platform exposes a unified, highly optimized HTTP REST interface to integrate advanced local RAG capabilities directly into external microservices or frontends.

GET/api/health•Check System Health

Returns connectivity and status metrics for Ollama local inference services, ChromaDB collection layers, document volume, and system uptime.

Example Request
curl http://localhost:8000/api/health
Example Response
{
  "status": "ok",
  "ollama": {
    "status": "connected",
    "message": "Ollama is running"
  },
  "chromadb": {
    "status": "connected",
    "message": "1 collection(s)"
  },
  "total_documents": 12,
  "total_chunks": 348,
  "uptime_seconds": 1845.2
}
GET/api/strategies•List Reasoning Strategies

Lists all available search and reasoning generation strategies supported by the platform, including their configured icons and details.

Example Request
curl http://localhost:8000/api/strategies
Example Response
{
  "strategies": [
    {
      "id": "adaptive",
      "name": "Adaptive",
      "description": "Automatically selects the best strategy based on query complexity and available context.",
      "icon": "🧠",
      "is_default": true
    },
    {
      "id": "naive",
      "name": "Naive RAG",
      "description": "Simple retrieve-and-generate pipeline. Fast and effective for straightforward questions.",
      "icon": "⚡",
      "is_default": false
    }
  ]
}
GET/api/collections•List Collections

Retrieves all vector search collections currently defined inside ChromaDB, including active document and chunk totals.

Example Request
curl http://localhost:8000/api/collections
Example Response
{
  "collections": [
    {
      "name": "default",
      "document_count": 5,
      "chunk_count": 142,
      "created_at": "2026-05-25T10:00:00Z",
      "description": "Default workspace collection"
    }
  ]
}
POST/api/collections•Create Collection

Initializes a new vector collection index. Used to isolate documents into distinct project spaces or domains.

Request Parameters
ParameterTypeRequiredDescription
namestringREQUIREDUnique URL-safe identifier for the new collection.
descriptionstringoptionalOptional summary of what this collection houses.
Example Request
curl -X POST http://localhost:8000/api/collections \
  -H "Content-Type: application/json" \
  -d '{"name": "financial-reports", "description": "Q3-Q4 Internal Audit Sheets"}'
Example Response
{
  "name": "financial-reports",
  "document_count": 0,
  "chunk_count": 0,
  "created_at": "2026-05-25T10:08:24Z",
  "description": "Q3-Q4 Internal Audit Sheets"
}
DELETE/api/collections/{name}•Delete Collection

Deletes a collection and all associated document vector chunk entries from disk. Note: The 'default' collection cannot be deleted.

Request Parameters
ParameterTypeRequiredDescription
namestring (path)REQUIREDName of the target collection to erase.
Example Request
curl -X DELETE http://localhost:8000/api/collections/financial-reports
Example Response
// Status 204 No Content
GET/api/documents•List Documents

Lists all normalized source files uploaded and chunked. Supports optional collection filtering.

Request Parameters
ParameterTypeRequiredDescription
collectionstring (query)optionalFilter documents belonging to a specific collection. Defaults to all collections.
Example Request
curl "http://localhost:8000/api/documents?collection=default"
Example Response
{
  "documents": [
    {
      "id": "e2a89c42-8f92-4f81-9b63-c7829d20c309",
      "filename": "Aetheris_Overview.pdf",
      "chunk_count": 18,
      "ingested_at": "2026-05-25T08:15:32Z",
      "collection": "default",
      "file_size": 1048576
    }
  ],
  "total": 1
}
DELETE/api/documents/{doc_id}•Delete Document

Permanently deletes a document and all corresponding chunks from the vector store, then triggers a background re-indexing of the sparse BM25 retriever index.

Request Parameters
ParameterTypeRequiredDescription
doc_idstring (path)REQUIREDThe UUID identifier of the document to remove.
collectionstring (query)optionalThe collection context. Defaults to the active connection.
Example Request
curl -X DELETE "http://localhost:8000/api/documents/e2a89c42-8f92-4f81-9b63-c7829d20c309?collection=default"
Example Response
// Status 204 No Content
POST/api/ingest•Ingest Documents

Accepts raw file uploads (PDF, DOCX, Markdown, Code), runs character extractors, executes the ONNX OCR fallback if necessary, generates contextual overlays, and indexes chunks into ChromaDB.

Request Parameters
ParameterTypeRequiredDescription
filesFile[] (multipart)REQUIREDArray of files to upload.
collectionstring (form)optionalTarget collection to save inside. Defaults to 'default'.
Example Request
curl -X POST http://localhost:8000/api/ingest \
  -F "files=@/path/to/handbook.pdf" \
  -F "collection=default"
Example Response
{
  "message": "Processed 1 file(s)",
  "files_processed": 1,
  "chunks_created": 34,
  "time_taken_ms": 4250.67,
  "errors": []
}
POST/api/query•Execute Standard Query

Performs synchronous hybrid retrieval and LLM response generation. Best for non-UI integrations or simple automated checks.

Request Parameters
ParameterTypeRequiredDescription
querystringREQUIREDThe user query or prompt.
strategystringREQUIREDReasoning strategy (adaptive, naive, hyde, corrective, self_rag, agentic).
collectionstringoptionalChromaDB target collection. Defaults to 'default'.
chat_historyarray (objects)optionalList of conversation messages: [{'role': 'user', 'content': '...'}, {'role': 'assistant', 'content': '...'}] for context-aware conversations.
Example Request
curl -X POST http://localhost:8000/api/query \
  -H "Content-Type: application/json" \
  -d '{
    "query": "What is the token limit of Aetheris?",
    "strategy": "adaptive",
    "collection": "default",
    "chat_history": []
  }'
Example Response
{
  "answer": "According to the Aetheris specifications, the default token configuration allows a context window of 8,192 tokens.",
  "sources": [
    {
      "document_id": "e2a89c42-8f92-4f81-9b63-c7829d20c309",
      "filename": "Aetheris_Overview.pdf",
      "chunk_text": "The LLM context limits configure context windows to 8192 tokens...",
      "relevance_score": 0.9412
    }
  ],
  "strategy_used": "naive",
  "timing": {
    "total_ms": 1120.45
  }
}
POST/api/query/stream•Stream Generative Query (SSE)

Streams tokens, citations, and system execution traces (agent step reflections) in real-time using Server-Sent Events (SSE). Preferred for production UI chats.

Request Parameters
ParameterTypeRequiredDescription
querystringREQUIREDThe user query or prompt.
strategystringREQUIREDReasoning strategy (adaptive, naive, hyde, corrective, self_rag, agentic).
collectionstringoptionalChromaDB target collection. Defaults to 'default'.
chat_historyarray (objects)optionalList of conversation messages for context-aware conversations.
Example Request
curl -X POST http://localhost:8000/api/query/stream \
  -H "Content-Type: application/json" \
  -H "Accept: text/event-stream" \
  -d '{
    "query": "Explain semantic chunking",
    "strategy": "corrective"
  }'
Example Response
event: trace
data: {"step": "grader", "message": "Evaluating retrieved vectors..."}

event: token
data: {"token": "Semantic "}

event: token
data: {"token": "chunking "}

event: sources
data: {"sources": [{"document_id": "doc-123", "filename": "chunk_methods.md", "relevance_score": 0.88}]}

event: done
data: {"timing": {"total_ms": 2341.1}, "strategy_used": "corrective"}
